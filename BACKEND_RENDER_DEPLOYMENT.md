# nmoo API Render Deployment

## Render Blueprint

Use the root `render.yaml` file to create the backend service and PostgreSQL database.

Expected resources:

```txt
Web service: nmoo-api
Database: nmoo-postgres
Database name: nmoo_db
Health check: /api
```

## Build And Start

Build command:

```bash
npm install && npm run deploy:prepare
```

Start command:

```bash
npm run start:prod
```

## Environment

The Blueprint sets:

```env
DATABASE_URL=<from nmoo-postgres>
JWT_SECRET=<generated by Render>
JWT_EXPIRES_IN=1d
NODE_ENV=production
FRONTEND_URL=https://nmoo.com
CORS_ORIGINS=https://nmoo.com
SWAGGER_ENABLED=false
RATE_LIMIT_TTL_SECONDS=60
RATE_LIMIT_MAX_REQUESTS=120
STRIPE_SECRET_KEY=<set in Render manually>
STRIPE_WEBHOOK_SECRET=<set in Render manually>
STRIPE_CURRENCY=sar
PUBLIC_ASSET_BASE_URL=https://cdn.nmoo.com
```

After the service is deployed, verify:

```bash
npm run env:check
npm run db:check
```

## API Checks

Open:

```txt
https://api.nmoo.com/api
```

Expected response:

```json
{
  "name": "nmoo نمو API",
  "status": "ok",
  "version": "0.0.1"
}
```

In production, API docs are closed unless `SWAGGER_ENABLED=true`:

```txt
https://api.nmoo.com/api/docs
```

Expected default result:

```txt
404
```

## Domain

Add `api.nmoo.com` to the Render service domains, then create the DNS record requested by Render.
